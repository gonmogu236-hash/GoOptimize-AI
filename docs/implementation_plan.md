# GoOptimize AI - 実装計画書

囲碁棋譜を解析し、弱点の分類・スコアリング・学習プラン生成を行うWebサービスのMVP構築計画。

## 主要機能
- **SGFパース**: 棋譜データの読み込み
- **フェーズ分類**: 布石・中盤・ヨセの自動判定（手数の3分割）
- **スコア評価**: 6つの評価軸（布石力、中盤戦闘力、死活精度、ヨセ精度、安定性、判断力）によるスコアリング
- **弱点検出**: 各フェーズおよびプレイスタイルに基づく弱点の特定
- **自動診断**: AI（ルールベース＋将来的にLLM）による診断レポート作成
- **学習プラン**: 個々の弱点に合わせた1週間・1ヶ月の最適化プラン提案

## 技術スタック
- **Frontend**: Vite + React, React Router, Canvas (Charts/Board)
- **Backend**: Express.js, Multer (File Upload)
- **Database**: SQLite (sql.js) → デプロイ時にSupabase (PostgreSQL) へ移行
- **Styling**: Vanilla CSS (Premium Dark Theme)

## 将来的なAPI差し替えガイド

### 1. KataGo APIへの差し替え
現在 `scoreEngine.js` でシミュレーションしている勝率と推奨手を、本物のAI解析に置き換える計画です。

**必要なインフラ**:
- WebSocket対応のジョブキュー（解析に分単位の時間がかかるため）
- GPU搭載サーバー（AWS EC2 g4dn系など）、または外部KataGo解析APIプロバイダとの契約

**実装手順**:
1. クライアントからSGFを受け取る。
2. ExpressサーバーからKataGo搭載のパースサーバーへジョブを送る。
3. 解析進捗をWebSocketやポーリングでクライアントに返す。
4. KataGoから返る `winrate` (勝率), `scoreLead` (目差), `visits` (探索数), `pv` (変化図: 推奨手数手) のJSONをパースする。
5. 現行のアルゴリズムを実戦データに基づく閾値判定に変更する。

### 2. LLM（OpenAI API 等）への差し替え
現在 `diagnosticGenerator.js` でテンプレート化している文章を、より柔軟な「熱いコーチング」へ進化させます。

**必要なインフラ**:
- OpenAI APIキー (GPT-4o-mini など)。従量課金が発生するため、これを行う際は上記のマネタイズ（有料版）と組み合わせるのが必須です。

**実装手順**:
1. レーダーチャート・弱点一覧・プレイスタイル・大悪手箇所などのデータをJSONに変換。
2. 「あなたはプロの厳しいが熱心な囲碁コーチです...」というプロンプトと共にJSONをLLMに投げる。
3. 安定した形式（Markdown等）で結果を得るため、Structured Outputs設定で行う。
4. 生成されたコーチング文をクライアント側へ返す。
